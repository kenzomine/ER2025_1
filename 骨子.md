# GitHub Copilot 活用ガイド  
## 議事録作成・会議分析編（骨子）

---

## 1. はじめに

### 目的
GitHub Copilotが「コーディング専用」ではなく、  
**日常的なドキュメント作成や会議の振り返りにも強力なツール**であることを伝える。

### 対象読者
- GitHub Copilotを利用可能だが  
  コード生成以外での活用方法を知らないソニーグループ社員

### メリット
- 議事録作成時間の短縮  
- 議論の構造化による理解促進  
- ネクストアクションの明確化  
- **会議内容の多角的分析**  
  - 論点整理  
  - 参加者の立場の可視化  
  - 議論の傾向把握  

### 選択箇所の修正機能
VS Code上でテキストを選択し、Copilot Chatに  
「選択範囲を○○してください」と依頼することで、

- ピンポイント修正
- 加筆
- 言い換え  

が可能。  
全文を書き直さず、**気になる箇所だけを効率的にブラッシュアップ**できる。

### LLM（大規模言語モデル）との協調作業という考え方
Copilotは  
**「完璧な成果物を一発で出す魔法のツール」ではない。**

- まず粗いドラフトを出させる  
- 気になる点を指摘  
- 質問・修正を繰り返す  

という**対話的プロセス**が効果的。

人間側が  
- 「どこが違うか」  
- 「何を補足したいか」  

を明確にフィードバックすることで、出力品質が向上する。

## 2. なぜ GitHub Copilot を議事録に使うのか？

### 従来の課題
- 録画・録音を見返すのに時間がかかる  
- メモが散乱し、要点をまとめるのが大変  
- 参加者以外に内容を共有するための清書に工数がかかる  

### Teams Copilot（Microsoft 365 Copilot）の限界
- 要点のみの出力で、**文脈や背景情報が欠落**しがち  
- 会議終了後でないと結果を確認できず、**リアルタイム活用ができない**  
- 出力形式やフォーカスポイントを細かく指定できない  
- 追加質問や深掘りがしにくく、一方向的な要約になりがち  

### Copilot による解決策
- 自然言語処理能力により、**雑多なテキストから文脈を理解して要約**  
- 「決定事項」「To-Do」など、**特定情報の抽出が得意**  
- 対話形式（チャット）で、気になった点だけを深掘りできる  

---

## 3. なぜ GitHub Copilot をプロジェクト分析に使うのか？

### 従来の課題
- 数ヶ月分の議事録やログを人間が読み返すのは物理的に不可能  
- 担当者の記憶や主観に依存し、客観的評価が難しい  
  - 「頑張った」で終わってしまう  
- 複数の会議体  
  - PJ6  
  - 文化祭  
  - WG 定例  
  にまたがる因果関係が見えにくい  

### Teams Copilot（Microsoft 365 Copilot）の限界
- 単一会議の要約は得意だが、**長期間・複数ファイル横断分析ができない**  
- 「PMBOKの視点で」「辛口で評価して」など、  
  **特定フレームワークを指定した高度な分析が難しい**  
- 大量テキスト（数ヶ月分のログ）を一度に扱えない場合がある  

### Copilot による解決策
- **大量ログの横断分析**  
  - 数ヶ月分の会議ログを一気に読み込み  
  - 全体像や傾向を把握可能  

- **フレームワークの適用**  
  - PMBOK  
  - 特定の評価軸（QCDなど）  
  を指定することで、専門的視点での構造分析が可能  

- **客観性の担保**  
  - 感情や忖度なし  
  - ログに基づく Fact から課題抽出  
  - 納得感のある振り返りができる

## 4. 準備・環境

### 必要なもの
- Visual Studio Code（VS Code）
- GitHub Copilot 拡張機能（および Chat 機能）
- 会議の記録テキスト  
  - Teams のトランスクリプト  
  - または自分が取ったラフなメモ  

### セットアップ
- VS Codeで会議メモのファイル（`.txt` や `.md`）を開いた状態にする

---

## 5. 基本的な使い方：議事録の自動生成

### 手順
1. **素材の用意**  
   - 会議のメモやトランスクリプトを VS Code で開く  

2. **Copilot Chat の起動**  
   - サイドバーから Chat を開く  

3. **プロンプト（指示）の入力**  
   - 例  
     - 「開いているファイルの内容を要約してください」  
     - 「この会議の決定事項とネクストアクションを箇条書きで抽出してください」  

4. **出力の確認と修正**  
   - 生成されたテキストをコピー  
   - 必要に応じて微修正  

### 使えるプロンプト例
- 「この会議の主要な論点を3つ挙げてください」
- 「AさんとBさんの意見の対立点をまとめてください」
- 「以下のフォーマットで議事録を出力してください  
  [日時][参加者][議題][決定事項][To-Do]」

---

## 6. 応用的な使い方：会議の分析と構造化

### 議論の構造化
- 時系列ではなく、**「トピック別」**に整理させる  
- 発散した議論を  
  - 「課題」  
  - 「原因」  
  - 「解決策」  
  のフレームワークで整理させる  

### 感情・雰囲気の分析（参考）
- 「会議の雰囲気はどうでしたか？」  
- 「懸念されている点はありますか？」  

と問いかけることで、  
**テキストから読み取れるニュアンスを言語化**できる。

### 不明点の解消
- 「〇〇という専門用語が出てきますが、この文脈ではどういう意味ですか？」  
と質問する。

## 7. 注意点とコツ

### ハルシネーション（嘘）への対策
- Copilotはもっともらしい嘘をつくことがある  
- **必ず元のメモと照らし合わせて事実確認**を行う  

### コンテキストの範囲
- 長すぎる会議ログは  
  - 分割して読み込ませる  
  - 重要部分を選択して質問する  

### セキュリティ
- 社内規定に従い、機密情報の取り扱いに注意  
- Enterprise版の仕様を確認するなど配慮する

## 8. まとめ
GitHub Copilotは  
**「対話型の優秀なアシスタント」**。

まずは  
**手持ちのメモを要約させるところから**  
始めてみることを推奨する。
